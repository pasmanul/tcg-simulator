# Web版 受け入れ基準 — ローカルアプリ同等動作テスト

> **目的**: tcg-simulator の Web 版（React + Vite + Konva）が、Python/PyQt6 製ローカルアプリと同等の動作をすることを検証する。
>
> **タグ凡例**
> - `[VERIFIED]` — ブラウザで動作確認済み
> - `[CODE]` — コードは存在するが動作未確認（テスト実施時に `[VERIFIED]` へ昇格させること）
> - `[PENDING]` — 未実装（将来の実装ガイドを兼ねる）
> - `[AUTO]` — Playwright 等で自動化推奨
> - `[MANUAL]` — 目視・操作確認が必要

---

## CAT-01: セットアップ / ファイルシステムアクセス

### コアフロー

```
Given ブラウザで index.html を開いた
When  SetupDialog でカードフォルダを選択する
Then  cards.json が読み込まれ、カード画像がキャッシュされること
```

### チェックリスト

| ID | 操作 | 期待結果 | 状態 | 自動化 |
|----|------|---------|------|--------|
| AC-01-01 | 初回アクセス時にセットアップダイアログが自動表示される | SetupDialog が開き、フォルダ選択を促すこと | [CODE] | [AUTO] |
| AC-01-02 | 「フォルダを選択」ボタンを押す | OS のフォルダピッカーダイアログが開くこと | [CODE] | [MANUAL] |
| AC-01-03 | cards.json を含むフォルダを選択する | cards.json が読み込まれ、カード一覧が表示されること | [CODE] | [AUTO] |
| AC-01-04 | cards/ サブフォルダ内の画像ファイルが読み込まれる | カードサムネイルが表示されること（back.jpg が裏面に使われること） | [CODE] | [MANUAL] |
| AC-01-05 | ページをリロードする | IndexedDB に保存されたディレクトリハンドルが復元され、権限再確認ダイアログが表示されること | [CODE] | [MANUAL] |
| AC-01-06 | 権限再確認ダイアログで「許可」を押す | フォルダが再接続され、カード画像が再表示されること | [CODE] | [MANUAL] |
| AC-01-07 | 権限再確認ダイアログで「拒否」を押す | セットアップダイアログに戻ること | [CODE] | [AUTO] |
